About the project

Projet récapitulatif - Machine Learning

Contexte du projet

Vous travaillez pour une entreprise qui crée des rééditions d’albums de musique légendaires au format vinyle. Le département marketing a une idée pour mettre en avant les services de votre entreprise et cherche à savoir si c’est réalisable. Cette idée consiste à publier une application web appelée “Les gens n’ont plus de goût”. Cette application permettrait de répondre à une question simple: “En se basant sur les caractéristiques musicales d’un morceau, est ce qu’un tube sorti il y a des années et qui est devenu un classique aurait connu le même succès s’il avait été sorti récemment”. Plus prosaïquement, l’idée est d'étudier le lien entre les caractéristiques musicales des morceaux sortis en 2022 et leur popularité pour appliquer ce modèle à n’importe morceau de l’histoire de la musique et en particulier du catalogue de votre entreprise.

Avant de vous lancer dans un tel développement, vous devez vous assurer de la faisabilité du projet. Cela consiste principalement en deux choses: Vous assurez de l’existence et de la disponibilité des données pour réaliser cette étude. Votre manager vous guide vers l’API du service spotify qui pourrait être une bonne première piste à explorer Étudier si un modèle à peu près convenable est réalisable. Si on suppose un score de popularité entre 0 et 100, il faudrait un modèle avec une MAE maximum d'environ 20.

Vous avez maximum trois jours pour valider ces deux points. Ces trois jours correspondent donc à une preuve de concept

Partie 2 :

Bravo, votre modèle semble prometteur, vous pouvez continuer à récolter plus de données et à finetune vos modèles. Quand vous serez satisfait de votre modélisation, vous pourrez passer à la mise en production.

Voici les modalités à respecter :

  • Une API pour votre modèle
  • Une app web en Django pour consommer votre API (authentification, page de profil, header, footer, plusieurs pages)
  • Une base de données SQL (table utilisateur, table des musiques)

Vous utiliserez docker pour transformer en containers chacune de ces composantes. Vous devrez utiliser docker compose pour votre app web et votre BDD (vous pouvez également inclure votre API mais comme elle va communiquer avec votre app par le bias d'internet, c'est facultatif). L'objectif final sera de déployer tous ces éléments sur Azure (App Services).

Votre application doit permettre à un utilisateur de rentrer la musique de son choix et d'avoir une prédiction qui répondra à la problématique suivante : "Est-ce que cette chanson aurait été populaire si elle était sortir en 2022". Vous êtes libre de formatiliser la prédiction comme vous voulez. N'oubliez pas que cette application est destinée à un utilisateur final, vous devrez donc pour toutes les mesures nécessaire pour rendre cette application ludique.
